# API Documentation

Base URL (when running via Docker Compose): `http://localhost:8000`

Interactive documentation (Swagger UI, auto-generated by FastAPI) is
also available at `http://localhost:8000/docs`.

---

## GET /health

Simple health check to confirm the backend is running.

**Response** `200 OK`
```json
{ "status": "ok" }
```

---

## GET /options

Returns the valid values for every form field, generated dynamically
from the dataset. The frontend uses this to populate its dropdowns.

**Response** `200 OK`
```json
{
  "job_roles": ["AI Engineer", "Computer Vision Engineer", "..."],
  "specializations": ["Analytics", "Computer Vision", "..."],
  "experience_levels": ["Entry", "Mid", "Senior", "Lead"],
  "countries": ["Australia", "Brazil", "..."],
  "industries": ["Automotive", "Consulting", "..."],
  "work_modes": ["Hybrid", "Onsite", "Remote"]
}
```

## POST /match

Scores all eligible jobs against the candidate's preferences and
returns the top 10 matches with explanations and ML salary estimates.

**Request body** (`application/json`)

| Field | Type | Required | Description |
|---|---|---|---|
| `job_role` | string | yes | Desired role (must be one of `/options` job_roles) |
| `ai_specialization` | string | yes | Desired specialization |
| `experience_level` | string | yes | `Entry`, `Mid`, `Senior` or `Lead` |
| `country` | string | no | Filter: only jobs in this country |
| `industry` | string | no | Filter: only jobs in this industry |
| `work_mode` | string | no | Filter: `Hybrid`, `Onsite` or `Remote` |
| `min_salary` | number | no | Filter: only jobs paying at least this (USD) |

**Example request**
```json
{
  "job_role": "Data Scientist",
  "ai_specialization": "LLM",
  "experience_level": "Entry",
  "country": "India"
}
```

**Response** `200 OK`
```json
{
  "count": 10,
  "results": [
    {
      "id": 86253,
      "job_role": "Data Scientist",
      "ai_specialization": "LLM",
      "experience_level": "Entry",
      "country": "India",
      "industry": "Automotive",
      "work_mode": "Onsite",
      "company_size": "Enterprise",
      "salary_usd": 32653,
      "salary_estimate": 34120,
      "salary_vs_market": -4,
      "match_percent": 100,
      "matched": [
        "Role: Data Scientist",
        "Specialization: LLM",
        "Experience level: Entry"
      ],
      "gaps": []
    }
  ]
}
```

**Response field notes**

- `match_percent` — weighted score: role 50%, specialization 30%,
  experience closeness 20% (see README for the formula)
- `matched` / `gaps` — human-readable explanations of what did and
  didn't match
- `salary_estimate` — the ML model's predicted market salary for a job
  with these attributes
- `salary_vs_market` — how this job's actual salary compares to the
  estimate, in percent (negative = below market)
- Jobs requiring experience more than one level above the candidate
  are excluded before scoring
- Results are sorted by `match_percent` descending, tie-broken by
  `salary_usd` descending
- If no jobs survive the filters, `count` is `0` and `results` is `[]`

**Notes**
- The first `/match` call after startup takes up to ~30 seconds while
  the salary model trains; subsequent calls respond immediately.
- Validation errors (missing required fields, wrong types) return
  `422 Unprocessable Entity` with details (FastAPI default).

## GET /model-info

Returns the salary model's evaluation metrics, computed on a held-out
20% test set at training time.

**Response** `200 OK`
```json
{ "mae": 10978.03, "r2": 0.887 }
```

- `mae` — mean absolute error of salary predictions, in USD
- `r2` — coefficient of determination (fraction of salary variance
  explained by the model)
